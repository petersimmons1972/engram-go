# Chart Rendering Regression - 2026-02-06

## Incident Summary

**Symptom**: Charts present in v030-v050 (10 charts) disappeared in v054+ (0 charts)

**Impact**: Pipeline claimed "charts_loaded: 16" but HTML had 0 visible charts. Reports shipped as READY_TO_SELL with no visual content.

**Customer Quote**: "You really did have them in previous builds. They were there and they looked awesome because I raved about them. I was proud of you. Righteously so."

## Root Causes (3 separate issues)

### 1. Section ID Mismatch (Configuration Error)

**Problem**: `SECTION_CHART_MAP` in `stage_5.py` had hardcoded section slugs that didn't match actual HTML generated by prose agent.

```python
# WRONG (in code):
'deployment-timeline-expectations': ['deployment_timeline.svg']

# ACTUAL (in HTML):
<h2 id="deployment">
```

**Why it happened**: Section headings evolve as prose generation improves. Hardcoded map never updated to match.

**Fix**: Updated map to match actual section IDs from generated HTML.

**Prevention**:
- Make section mapping dynamic OR
- Add validation that checks SECTION_CHART_MAP keys exist in actual HTML
- Fail loudly if chart can't find its target section

### 2. Duplicate Method Definition (Python Silent Failure)

**Problem**: `_abbreviate_vendor_name()` defined twice in `chart_generator.py`:
- Line 101: `def _abbreviate_vendor_name(self, name: str, max_chars: int = 20)`
- Line 804: `def _abbreviate_vendor_name(self, name: str)` (no max_chars)

Second definition silently overwrote first. Code calling `_abbreviate_vendor_name(..., max_chars=18)` crashed with TypeError.

**Why it happened**: Refactoring left duplicate. Python doesn't warn about redefinition.

**Fix**: Removed duplicate definition.

**Prevention**:
- Use linting tools that detect duplicate definitions
- Search for `def method_name` before adding new methods
- Code review to catch duplicates

### 3. BeautifulSoup Destroys SVG Tags (Post-Processing Bug)

**Problem**: `_ensure_text_density_breaks()` used BeautifulSoup to inject visual breaks. BeautifulSoup (ALL parsers: html.parser, lxml, html5lib) mangles SVG tags:

```html
<!-- BEFORE BeautifulSoup -->
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 600 450">

<!-- AFTER BeautifulSoup -->
<svg xmlns="http://www.w3.org/2000/svg" viewbox="0 0 600 450">
```

Lowercases `viewBox`, strips or modifies xmlns attributes. Broke chart rendering.

**Why it happened**:
- BeautifulSoup is HTML parser, not XML parser
- SVG uses XML namespaces which HTML parsers don't handle correctly
- Even "html5lib" which claims to preserve tags still modifies attributes

**Fix**: Rewrote `_ensure_text_density_breaks()` using pure regex instead of BeautifulSoup.

**Prevention**:
- **NEVER use BeautifulSoup when SVG tags are present**
- Use regex for HTML manipulation when preserving exact structure is critical
- Test post-processing functions with SVG content

## Lessons Learned

### What Worked Well

✅ **Systematic debugging**: Traced data flow through all stages
✅ **Intermediate verification**: Checked charts at each processing step
✅ **Debug logging**: Added counters to track SVG tags through pipeline
✅ **Not giving up**: Charts were generating, just being destroyed later

### What Went Wrong

❌ **Trusting intermediate metrics**: "charts_loaded: 16" gave false confidence
❌ **No visual validation gate**: Pipeline shipped reports without verifying charts visible
❌ **BeautifulSoup assumption**: Assumed all parsers handle HTML safely
❌ **Silent failures**: Duplicate method definition didn't raise error

### Critical Insights

1. **Verify what customer sees, not intermediate state**
   - charts_loaded ≠ charts_rendered
   - Generate ≠ Inject ≠ Survive post-processing

2. **Post-processing destroys content**
   - Any HTML manipulation after injection can break things
   - BeautifulSoup is NOT safe for SVG-containing HTML
   - Use regex when exact preservation required

3. **Configuration drift**
   - Hardcoded section maps decay as prose evolves
   - Need dynamic matching OR validation that map is current

4. **Python gotchas**
   - Duplicate method definitions silently overwrite
   - Last definition wins, no warning

## Prevention Strategy

### Validation Gates Needed

1. **Gate: Chart Count Validation**
   ```python
   svg_count = html.count('<svg')
   if svg_count < 4:
       fail("CRITICAL: <4 charts rendered")
   ```

2. **Gate: Visual Diff Testing**
   - Render HTML to image
   - Count visual elements
   - Ensure charts actually display

3. **Gate: Section Mapping Validation**
   ```python
   for section_id in SECTION_CHART_MAP.keys():
       if f'id="{section_id}"' not in html:
           fail(f"Section {section_id} not found - chart cannot inject")
   ```

### Code Patterns

**DO**:
- ✅ Use regex for HTML manipulation when SVG present
- ✅ Verify final output matches expectations
- ✅ Add intermediate checkpoints with counters
- ✅ Test with actual generated content, not mocks

**DON'T**:
- ❌ Use BeautifulSoup when SVG tags are in HTML
- ❌ Trust intermediate success metrics
- ❌ Hardcode section mappings that can drift
- ❌ Skip visual verification before shipping
