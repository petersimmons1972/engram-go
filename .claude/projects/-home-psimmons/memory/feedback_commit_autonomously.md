name: Commit autonomously after completing work
description: Don't ask permission to commit after completing an implementation task — just do it
type: feedback
---

After completing implementation work, commit without asking. The user explicitly instructed autonomous bug fixing and the commit is a natural part of completing the task, not a separate decision requiring approval.

**Why:** User corrected me for asking "Want me to commit?" after completing all 7 batches of an implementation plan. Asking broke the flow and wasted time.

**How to apply:** When the user gives an explicit implementation plan and you complete it, stage and commit as part of the task. Only pause before commit if there's genuine uncertainty about what to include.
