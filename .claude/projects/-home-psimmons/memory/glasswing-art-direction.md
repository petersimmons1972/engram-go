# Glasswing Art Direction

## Primary Aesthetic: Retro-Futurism

The Glasswing series is about AI that behaves unexpectedly, regulatory frameworks built for a simpler problem, and the gap between the future we designed for and the one we're actually in. Retro-futurism carries this argument — the aesthetic of optimistic futures that didn't account for what actually happened.

Reference touchstones: NASA/Soviet space program technical drawings, 1960s-70s mission control dashboards, CRT terminal readouts, blueprint aesthetics, Lem-era sci-fi diagrams, vintage computing UI.

## Artwork Types

It doesn't always have to be original illustration. Valid artwork types:
- Data charts and graphs (when the data IS the argument)
- Technical diagrams (detection gap timelines, capability curves, architecture diagrams)
- Original illustrations (when a concept needs a visual metaphor)

Choose the type that serves the article's core argument best.

## Artist Selection

No default artist — pick based on what the article needs:
- **Greiman**: Layered information complexity, digital-era sophistication, data-rich diagrams
- **Cassandre**: Geometric precision, clean machine-age forms (significant overlap with retro-futurism)
- **Direct SVG chart work**: When the data visualization IS the header

Avoid defaulting to any single artist for the whole series.
