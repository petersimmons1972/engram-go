# Failure-Mode Standard (Universal)

**Scope:** All projects. Per-repo checklists INHERIT from this catalog and may append repo-specific checks.
**Process:** Every time a bug burns a cycle вҶ’ append a catalog row + a check entry in the relevant class below.
**Per-repo pattern:** `<repo>/bin/checkin-lint.sh` (automates рҹӨ– items) + `<repo>/docs/CHECKIN-CHECKLIST.md` (inherits + extends).
**First seeded:** 2026-06-13 from the aifleet embedder-identity and poller-relocation sagas.

## Bug Classes

### A. Identity vs. Routing

A routing/serving label (alias, tag, selector, role suffix) leaked into a field that a downstream system persists and compares as a **stable identity** вҶ’ false mismatch, blocked writes, or spurious reprocessing.

- рҹӨ– **No routing/role word encoded in a stable identity field.** Flag names containing `-live`, `-reembed`, `-bulk`, `-burst`, `-fast`, `-primary`, host names, ports, or role descriptors wherever they feed a persisted identity comparison.
- **The advertised name == the authoritative identity stored downstream.** If any system persists `<thing>_name` / `<thing>_id`, every producer must advertise the exact same string.
- **Same artifact вҮ’ same identity.** Two endpoints/versions serving identical artifacts MUST advertise the identical identity. Different name for the same artifact = a future false mismatch.
- **Different artifact вҮ’ deliberately different identity** (never reuse a name across distinct artifacts).
- **Role is expressed in routing, not in the name.** primary/bulk/fast/etc. = which endpoint/URL/queue a consumer targets, or an explicit routing field вҖ” never the artifact identity string.
- **No field carries two orthogonal meanings.** If one string is both the identity AND the routing selector, split them before merge.
- **A re-process/re-index operation fires ONLY on a real content-or-schema change or explicit force flag.** An identity rename or label change must short-circuit to no-op rather than triggering a mass re-process. Guard: check whether a genuine underlying change occurred before enqueueing work.

- рҹӨ– **Validate model name casing in ConfigMap matches exactly what the embedding backend registers**
### B. Install / Config Layout Integrity

A unit, manifest, or wrapper references a path that the installer does not actually produce вҖ” wrong filename, missing `.sh` extension, binary never vendored, or ordering race between `daemon-reload` and `enable`.

- рҹӨ– **Every `ExecStart`/`ExecStartPre`/wrapped path resolves to what the installer actually produces** (exact filename, including `.sh`-or-not, and exact location). Render with real substitutions; confirm the target exists post-install. Offline linters (`systemd-analyze verify`) cannot see runtime specifiers вҖ” verify against a staged install.
- **Every referenced binary/script is actually vendored AND installed.** Grep each unit's targets; confirm a source file exists in-repo and an install line places it at the expected path.
- **Wrappers that `exit 0` don't mask a failing child.** Any run/notify wrapper must propagate the real exit code.
- **`daemon-reload` before `enable`.** Enabling a unit before the daemon has reloaded the new unit file can self-disable a timer or cause stale-unit behavior. Sequence: write unit вҶ’ `daemon-reload` вҶ’ `enable` вҶ’ `start`.
- **`docker compose up` wrapper scripts must not shadow `.env` with inline `VAR="$(cmd)"` assignments.** Docker Compose prefers the calling shell's environment over the project `.env` file; an inline dynamic assignment (e.g. `HOSTNAME="$(hostname -f)"`) wins over the `.env` value even when `.env` is authoritative. Read `.env` explicitly and let the file value win: `_v="$(sed -n 's/^KEY=//p' .env)"; KEY="${_v:-$(fallback)}" docker compose up -d`. (FM-89)

- **Before editing a k8s Secret, inspect .metadata.managedFields[].manager / ownerReferences for 'externalsecrets'; if ESO-managed, edit the external source (Infisical) + force-sync the ExternalSecret, never the Secret directly.**
### C. Portability вҖ” No Machine/User Literals

Hardcoded user home paths, version-pinned tool paths, or specifier tokens used outside their valid expansion context break on any other user, machine, or upgrade.

- рҹӨ– **No `/home/<user>` literal in any unit/drop-in/installer/config.** Use `%h`, `__HOME__`, `__REPO_ROOT__`, or equivalent install-time tokens only. `rg -n '/home/[a-z]' .` must return zero hits.
- **Know where template specifiers expand.** `%h` expands in `ExecStart`, `ReadWritePaths`, etc. вҖ” NOT inside `Environment=` strings. Use install-time `sed` substitution tokens in `Environment=`.
- **No version-pinned absolute tool paths** (e.g. `~/.nvm/versions/node/vX.Y.Z/bin/...`). Resolve dynamically (`command -v`) or via install-time template вҖ” never pin a path that dies on the next tool upgrade.

- рҹӨ– **External service endpoints must be configurable via flag + env with a neutral or required-explicit default; grep for personal domains before publishing; reviewers should flag any capability that cannot be configured from docs alone.**
- **Every CLI --addr/--host/--port flag must have a corresponding env-var fallback registered via viper or equivalent**
### D. Fail-Loud вҖ” No Silent Masking

A missing dependency, empty result, failed child process, or degraded fallback is swallowed and reported as success вҖ” masking the failure for hours or weeks.

- **Critical/gating paths fail CLOSED, not open.** A missing dependency, empty result, or fallback must surface as a non-zero exit or an explicit error flag, never silently swallowed.
- **A `Result=success` that could mask a failed child is suspect.** Verify the leaf process actually ran and produced output.
- **Fallback/degraded modes must emit a warning-level signal,** not silently score as a normal result.
- **GPU host offline вҶ’ models silently absent.** When a watcher heartbeat window expires (host unreachable), the controller stops advertising that host's models. No alert fires by default. Diagnosis: `curl https://ai-fleet.petersimmons.com/registry | jq .` вҖ” look for stale `last_seen` timestamps before blaming the router or model. (FM-91)
- **`OLLA_EXCLUDED_HOSTS` is a temporary manual circuit breaker, not a permanent setting.** Track it with an issue number in a comment; remove it in the same PR that resolves the underlying problem. `grep OLLA_EXCLUDED_HOSTS olla.yaml` returning non-empty with no companion issue reference = stale breaker. (FM-90)

- рҹӨ– **Always use `set -e` at top of K8s Job scripts so any command failure propagates**
- рҹӨ– **Before releasing chunk claims or assuming reembed will pick up all chunks, verify the claim predicate covers the actual stuck state**
- **A liveness renewer must tolerate transient failures across the full TTL grace (ceil(TTL/interval) consecutive misses) before declaring loss, with a regression test injecting a transient mid-run failure.**
- **Route plan-feedback through a `comment_only` capability, not an artifact-expecting `review` capability; a dead-letter with "preserved unpushed work" or unpushed commits is the signal that the wrong capability was used.** The Codex CLI treats any non-trivial prompt on a `review` item as an implementation task and creates commits; if `git push` is allow-listed-blocked, the worktree is preserved but the item dead-letters silently. Pre-enqueue gate: capability=`review` requires a code-artifact deliverable; plan-feedback uses `comment_only`. (FM-108)
- **Cancel endpoint must be idempotent across all terminal states (done, dead-letter, cancelled); verify that POST /item/{id}/cancel returns 204 for any terminal state, not 409. Pre-deploy gate: integration test that asserts cancel on a dead-letter item succeeds.**
- **Every '# pragma: no cover' live binding has at least one real (non-faked) smoke test that actually invokes it; and after editing an import block in a formatter-hooked repo, grep-confirm the import survived the hook.**
- **An eval/measurement harness can score its own artifacts instead of the system under test; a single headline number with no per-stage instrumentation silently masks that the HARNESS, not the SUT, is failing.** A probe that selects from sanitized gold-only inputs, applies a strawman filter, or over/under-extracts produces a number that looks like signal (e.g. a "4.7% floor") but actually measures the harness вҖ” the conclusion looks earned and is not. Gate before trusting any harness number: (1) instrument per-stage failure CLASSES (extraction vs object-linking vs arithmetic vs gold-parse) so the dominant failure is visible; (2) add an upper-bound arm (e.g. no-object-filter) that isolates the layer under test from the linking artifact; (3) run drop/perturbation sensitivity; (4) RED-TEAM the harness itself. A metric with no upper-bound arm and no failure-class breakdown is not yet trustworthy as a go/no-go. (FM-112)
- **Verify each cobra flag has a corresponding struct field assignment and is exercised in at least one integration test**
### E. Thermal / Resource-Role Safety

A change to naming, routing, or configuration removes or bypasses a safety pin that restricts sustained/bulk workloads to hardware that can handle them.

- **Resource-role pinning survives identity/routing changes.** Constrained devices (fanless, low-memory, shared CPU) must never receive sustained/bulk load. If a change alters how a consumer selects a resource, confirm the safety pin still holds by an independent selector (URL, label, affinity) that cannot be removed by a rename.
- **Bulk workloads target the correct resource by a selector that survives a name collapse.**
- **Sequence routing changes resource-safely:** assign the device/resource pin BEFORE collapsing whatever selector currently routes to it.

- рҹӨ– **Run per-table DELETEs in separate autocommit statements; scale the live service to 0 first**
- рҹӨ– **Verify embedding endpoint returns 200 on /v1/models before scaling reembed or releasing claims**
### F. Repo Hygiene / Canonical Home

Infra or code living in the wrong repo, look-alike stubs installed in place of the real artifact, or dual install paths creating ambiguity about which version is live.

- **Artifacts live in their owning repo.** Do not add infra to a repo whose stated purpose is something else.
- **One canonical artifact вҖ” no look-alike stubs.** No stub that can be installed as the real implementation; no dual install paths for the same service.
- рҹӨ– **`git remote -v` matches the intended repo before editing.** A session's cwd may belong to a different repo or worktree than the one you mean to change вҖ” verify before every cross-repo edit.

- рҹӨ– **Never TRUNCATE a shared-project table; use scoped DELETE or dump/restore only the target project rows**
- рҹӨ– **Run `git fetch` + `git status -sb` before trusting recon of a local repo; verify claims against `git show origin/main:<path>` / `git ls-tree origin/main`, never the working tree alone.**
- рҹӨ– **Grep for conflict markers (`rg '^(<<<<<<<|=======|>>>>>>>|\|\|\|\|\|\|\|)'`) across tracked text files; treat any hit as a hard fail.**
### G. Deploy / Verify Discipline

Skipping staged verification before enabling a live artifact, or linting offline instead of against the live install output.

- **Prod deploys are staged and reversible.** Pattern: backup вҶ’ stage with `NO_ENABLE` (or equivalent dry-run) вҶ’ verify against live artifact вҶ’ enable вҶ’ auto-rollback on any verify miss.
- **Verify against the LIVE artifact, not just an offline lint.** Offline tools (`systemd-analyze`, schema linters) can pass an artifact that the live executor rejects. Staged install-layout checks catch what offline lints miss.
- **Prod deploys are founder-gated** (see `~/CLAUDE.md` В§Cost Guardrails).

- **A generation prompt that forces commitment to a specific answer must be tested against weak/conflicting/negated-evidence fixtures and provide an abstention path before being claimed a win.**
- **Any ranking/retrieval/metric change must report per-category/per-segment deltas and gate on no segment regressing past threshold, never aggregate-only.**
- **A round-trip canary must assert a TERMINAL done with a non-empty deliverable (not just successful claim/execute); the client report-body fields must match the server's required terminal-report contract.**
- **bats setup() must keep errexit ON (units-under-test invoked via run, which absorbs their status); a canary test ending in bare false must FAIL. Grep test setup files for 'set +e'.**
- **The codex-poll runner must fetch origin and rebase/reset its worktree to origin/<default-branch> immediately before creating a task branch; verify the branch's merge-base equals current origin/main HEAD before the run, and rebase-before-push.**
- **The brief MUST pin the invocation contract (daemon vs one-shot, env-config vs CLI args, entrypoint command); before cutover, run a deploy smoke-test that starts the component EXACTLY as the runtime does (no-args entrypoint) and confirms it stays up, not just that unit tests pass.**
- **Token rotation requires consumer lockstep вҖ” cycle ALL consumers in the same window.** After rotating `ATTACH_TOKENS` in Infisical: force-sync ESO вҶ’ confirm Secret `resourceVersion` bumped вҶ’ restart fleet-dispatch вҶ’ restart BOTH consumers (Hermes container + codex-attach.service on the codex node) before the old token is invalidated. Skipping any consumer causes silent 401 work-stall. Runbook: `~/docs/runbooks/fleet-dispatch-token-rotation.md`. (FM-97)
- рҹӨ– **Never restart a deployment before confirming the ESO-managed Secret `resourceVersion` has changed.** Restarting before ESO commits the new value loads the old secret into new pods. Poll `kubectl get secret <name> -n <ns> -o jsonpath='{.metadata.resourceVersion}'` until it increments. (FM-86)
- **A capability must have exactly ONE intended consumer; assert `lease_holder` matches the expected consumer after every enqueue.** Routing two consumer classes to the same capability produces non-deterministic claim order and masked dead-letters; the fence/retry recovery is not a substitute for deterministic routing. Enumerate all tokens' Consumes[] lists and flag any overlap before merge. (FM-107)
### H. Ephemeral Resource Leakage вҶ’ Host Exhaustion (GLOBAL RISK)

**Universal directive (founder, 2026-06-14): any testing that creates ephemeral infrastructure MUST clean up after itself вҖ” Docker *and* Kubernetes, and any other ephemeral runtime.** The agent that created the resource owns its teardown; "the test passed" is not done until the resources it spun up are gone.

Ephemeral test/build/dev artifacts (containers, **anonymous volumes**, images, build cache; k8s test namespaces, PVCs, pods, jobs, LoadBalancers) accumulate on a **shared host/cluster** until they exhaust disk or memory and crash *unrelated* workloads. On the workstation this OOMs the **COSMIC desktop** (and any co-located containers) вҖ” a known failure mode. On a cluster, orphaned PVCs and test namespaces exhaust storage/quota and strand finalizers. The agent that ran the tests is responsible for the cleanup.

- **Clean up every Docker resource you create for a test/build.** After any `go test` with testcontainers, `docker build`, or ad-hoc `docker run`, remove the containers AND their **anonymous volumes**. Ryuk reaps containers but anonymous Postgres/data volumes routinely persist (observed: weeks of leaked volumes).
- **Clean up every Kubernetes resource you create for a test.** Tear down test namespaces (`kubectl delete ns <test-ns>` cascades pods/PVCs/jobs), and verify **PVCs are actually released** (PVCs can outlive their namespace's pods and pin backing storage; check `kubectl get pvc -A` for orphans, and watch for stuck `Terminating` finalizers). Prefer ephemeral namespaces per test run so teardown is a single delete. Never leave a test LoadBalancer/Ingress allocated. Same delta-only discipline: remove only what your run created, never blanket-delete a shared cluster's resources.
- рҹӨ– **Verify-after-test (delta-only):** snapshot `docker volume ls -qf dangling=true` before a run; after, remove **only the volumes your session created** (filter by `CreatedAt` = today/this session). Assert zero session-created dangling volumes remain.
- **NEVER global-force-prune a shared host.** `docker volume prune -f`, `docker system prune -a` on a multi-tenant box can destroy *other* workloads' detached data. Targeted `docker rm` / `docker volume rm <id>` of your own artifacts only. (The auto-mode classifier correctly blocks global prunes вҖ” do not work around it.)
- **Watch host headroom as the early signal.** `docker system df` reclaimable creeping into tens of GB, or dangling-volume count climbing, precedes the OOM. The desktop crashing is the *late* signal вҖ” by then it is already too late.
- **Long-running вү  wanted.** A container "Up N days" may be stale/forgotten (e.g. a gateway that "shouldn't be running"); confirm intent before assuming it's load-bearing, and stop reversibly (`docker stop`, not `rm`).

### I. Container Runtime вҖ” Design

A container runtime design assumption fails silently: schedulers never fire inside the container, inner sandboxes crash on modern kernels, stateful paths write to the ephemeral layer instead of a volume, or a topology assumption about service locality is wrong. Failures in this class were catalogued during the Codex containerisation pilot (FM-RC-01 through FM-RC-10).

- **Active liveness, not process-RUNNING.** Scheduler, timer, and periodic tasks inside a container must surface a last-successful-cycle timestamp to HEALTHCHECK; freshness is asserted, not just process presence.
- **Drop the inner sandbox when the container is the boundary.** `bwrap`/userns/seccomp inside a container inverts privilege probes on modern kernels. Assert confinement via `/proc/1/status` (fail when unconfined), never via sandbox-success.
- **All stateful paths on named volumes; startup asserts mounts.** Lease/heartbeat/cache/state paths that default to HOME write to the ephemeral layer and are lost on restart. Startup must verify every stateful path is volume-backed; a restart-persistence test must pass.
- **Single source of volume names; assert the running container mounts the expected set.** Installer-created vs compose-managed volumes diverge вҶ’ orphans + wrong mount.
- **Startup probe of each declared endpoint; no localhost/host-gateway assumption without verification.** A depended service assumed local may be LAN/remote; the startup probe catches topology mismatches before the first job.
- **Default-deny egress; hostname-filtering proxy re-resolves per request.** Static IP pins break when CDN/SaaS hosts rotate IPs; container-resolved вү  host-resolved. Test: allowed host must survive a simulated A-record change.
- **Poll-gate flag (default OFF) until cutover; mask host scheduler at cutover.** Prevents double-claim when both host and container scheduler run concurrently.
- **Supervisor restart + backoff + circuit-breaker; per-component health + last-run timestamps + tagged logs.** Single-container blast radius: one crash must not silently kill all co-located services.
- **Atomic credential write; documented rotation/revoke path; restart-persistence test.** Device-flow/bearer tokens on a volume are the crown-jewel: concurrent-write corruption or undocumented recovery path blocks all operations.

### J. Container Runtime вҖ” Operational

Live-discovered failures during container install: missing image assets, cross-script name drift, phantom supervisor paths, config parse crashes, healthcheck connectivity gaps, silent missing-secrets boot, and egress-rule persistence after reboot. Failures in this class were catalogued during the Codex pilot (FM-RC-11 through FM-RC-21).

- рҹӨ– **Test every runtime-referenced file exists at its installed path in the image.** Missing pinned-SHA config or script causes a silent cycle-1 abort.
- рҹӨ– **Every sibling-script invocation in the container resolves to an executable in the image (name vs name.sh).** Cross-script name/extension drift causes silent failure. Cross-ref: FM-04, FM-05 (same class, container-specific context; FM-RC-12 is not assigned a separate FM-ID as it is covered by the existing entries).
- рҹӨ– **Assert every supervisor `program command` path resolves; no `|| true` masking.** Phantom paths report RUNNING while doing nothing.
- рҹӨ– **Config-parse-reaches-RUNNING boot test.** `supervisord` format sigils (`%`) on a literal value crash on start вҖ” caught only by a real boot test, not a lint.
- рҹӨ– **Healthcheck actually returns healthy in the boot test.** Missing `[unix_http_server]` block means `supervisorctl` healthcheck is always unhealthy.
- рҹӨ– **`env_file required:true`; required-vars list complete (incl. memory/API keys).** `required:false` lets the container boot without secrets вҶ’ partial-credential run, no error.
- рҹӨ– **Installer aborts on any step failure; idempotent; genuine `--dry-run`.** Missing `set -e` in installer deploys a stale image after a failed build with no signal. Cross-ref: FM-18 (same fail-loud class; FM-RC-17 is the installer-script instance).
- **`nohup </dev/null` durable launch for device-flow login.** Detached login process killed on session teardown вҶ’ code expires unused.
- **Backoff wrapper, fresh code per retry, bounded by deadline for device-flow polling.** CLI polls token endpoint too hard вҶ’ self-429, login dies.
- рҹӨ– **Egress persistence hook ordered after `docker.service`; documented in runbook.** Runtime-only nft rules vanish on reboot вҶ’ silent open egress.
- рҹӨ– **Resolve the actual docker network name (compose-prefixed `<project>_<net>`); parameterize.** Script-default network name mismatch вҶ’ egress fail-closes or mis-scopes.
- рҹӨ– **Container context disables the Codex CLI inner bwrap sandbox; host context unchanged.** bwrap cannot create a user namespace inside cap_drop=ALL + no-new-privileges вҶ’ workspace-write tasks fail. Fix: gate `--dangerously-bypass-approvals-and-sandbox` on `CODEX_INNER_SANDBOX=off` (Dockerfile ENV only). Tests assert containerвҶ’bypass-on, hostвҶ’bypass-off. Cross-ref: FM-RC-31.

### K. Agent Runtime вҖ” Process / Coordination

Process failures at the human-agent coordination boundary: executor refuses a legitimate coordinator-relayed authorization, or a value resolved at apply time diverges from what the container sees at runtime. From FM-RC-22 and FM-RC-23.

- **Briefs must separate the human-only act (approval) from the non-authorization act (artifact generation).** An executor that conflates "issue a code" with "grant access" blocks real work while the human waits. Name what the executor may vs may not originate.
- **Resolve where the value is consumed, or re-resolve continuously.** Any value resolved on the host at apply time but used by the container at runtime can diverge (DNS, IPs, tokens). Generalisation of FM-RC-07 / FM-28.
- **Pre-authorize fleet socialization transports in settings.json; ABORT after the first permission-denied result вҖ” never retry.** Background/worktree agents have no interactive channel to approve Bash permission prompts; an allowlist entry for `ssh codex.petersimmons.com` / `ssh hermes.petersimmons.com` is required or the agent silently burns tokens for zero progress. (FM-75)
- рҹӨ– **Never use `-v`/`-sv`/`--verbose` flags (or equivalents) against authenticated endpoints in agent briefs.** Verbose HTTP modes write raw `Authorization: Bearer <token>` lines into the session transcript or CI log. Prefer `xh` (default non-verbose) or `curl -s` without `-v`. On discovery of a leaked token in a transcript: redact the log entry, rotate the token immediately via the rotation runbook (`~/docs/runbooks/fleet-dispatch-token-rotation.md`), confirm no copy persists in Engram or log archives. Deterministic grep check before archiving: `rg 'Authorization: Bearer|Bearer [A-Za-z0-9+/=-]{20,}' <transcript>`. (FM-96)
- **Two sessions/agents editing the same files on one working tree race; a phantom compile error that resolves on a clean re-read is the signal.** When concurrent writers (two Opus sessions, or a session and a parallel implementer) share one checkout/branch, edits interleave вҖ” a diagnostic can catch an intermediate state (e.g. `aggregate.go` references `f.MoneyRole` before `frame.go`'s definition lands) and report a "broken build" that is actually green on a fresh build вҖ” and either writer can silently clobber the other. Detection: a compile/diagnostic error that disappears on `go build` re-check; `git status` shows AM-staged files modified by the other writer; overlapping file mtimes from two sessions. Gate: ONE writer per file вҖ” assign file/dir ownership BEFORE parallel work, and put concurrent sessions in separate git worktrees; never two writers on one checkout of the same file. (FM-111)

### L. GitOps Declarative Drift

A live cluster fix (NetworkPolicy, Secret, deployment field, scale, pod interrupt) is applied via `kubectl apply/edit/scale` and verified working, but the change lives ONLY in the cluster, not in the source manifests at the GitOps-watched path. Later a GitOps reconcile (ArgoCD/Flux) or redeploy/rebuild reverts it, and the original problem silently returns.

- **ANY live cluster fix MUST be written into the source manifests at the GitOps-watched path and merged to the tracked branch.** A live-only change drifts from source and is reconciled away or dropped on rebuild. `kubectl apply` followed by "it works now" is NOT done вҖ” the fix must land in the repo on the watched branch.
- **Before applying live, check for GitOps reconciliation and External Secrets Operator.** ArgoCD (`kubectl get applications`), Flux (`kubectl get kustomizations`), and ESO-managed resources (inspect Secret `.metadata.managedFields[].manager`) all mean the resource is source-controlled. Edit the SOURCE (repo / Infisical), not the live object.
- **Verify the source path, not just the live object.** After a fix: read the source manifest on the watched branch to confirm the change is present. Use `git show origin/main:path/to/manifest.yaml` or `git ls-tree origin/main`, never the working tree or the live cluster state alone.
- **Gitops-reverted fixes are a frequent pattern.** Symptom: a "fixed" problem reappears after a reconcile cycle or node restart, with no change to the live cluster state visible during the window. Root cause: it was live-only. Check: `git log -p <path>` on the GitOps branch for the fix. If absent, it was never persisted.

### Case Study 1 вҖ” Embedder Identity vs. Role Leak (aifleet / Engram, 2026-06-12)

**What happened:** aifleet's olla per-role routing commit encoded each GPU's role into the olla `--alias` (`bge-m3-live`, `bge-m3-reembed`). Engram-go reads that alias as `project_meta.embedder_name` вҖ” a persisted identity it compares on every write. Because the alias no longer matched the stored identity `BAAI/bge-m3`, every `memory_store` returned `embedder_mismatch` and was blocked. The model weights were identical on both endpoints; only the advertised name differed. No re-embed was needed.

A secondary hazard: collapsing the names without first giving the bulk worker a URL-based selector would have sent a 259 GB bulk re-embed to the fanless MI-50, which has no active cooling.

**Class:** FM-01, FM-02, FM-03, FM-09.

**The checks that would have caught it:**
- Grep `--alias` values for role words (`-live`, `-reembed`, `-bulk`) before merging the routing commit. (рҹӨ–)
- Confirm the advertised alias == the value persisted in the downstream system's identity field. (Judgment, pre-merge)
- Before collapsing names, confirm bulk-workload routing uses a selector other than the name being changed. (Judgment, pre-deploy)

**Fix applied:** Three ordered prod steps вҖ” (1) fix primary alias to `BAAI/bge-m3` (stops the bleed), (2) give bulk worker a direct URL pin (thermal safety independent of name), (3) unify bulk alias. Full runbook: `~/.claude/worktrees/advisory-protocol-package/.claude/plans/RUNBOOK-embedder-identity-vs-role.md`.
